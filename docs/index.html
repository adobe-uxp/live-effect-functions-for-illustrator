Issue 1:

The applyEffect() method puts the live effect at the end of the Appearance, after the stroke and fill. However some effects need to be placed at the beginning, before the stroke and fill. In other respects the live effect is correctly applied, so that manually moving it from the bottom to the top of the Appearance Panel resolves the problem completely. Sadly, this makes automation useless!

Thanks to KurtGold at Adobe Support Community for working out how to fix PathFinder Add, Intersect and MinusFront, by adding a Pathfinder Divide Live Effect first.

Issue 2:

With the happy exception of Gaussian Blur, all the PS effects encode their parameters in an ascii85 string, accompanied by a dataSize property. that I haven't been able to usefully decode using free tools. And even if I managed to decode the parameters, I don't have a clue how to mark them up for the applyEffect method's XML.

(I haven't tried every single one of the filters that I've marked with Issue 2. Just the ones I wanted, eg. Radial Blur.)
